# 貓咪乾物質計算器 - 第二階段功能需求文檔 (PRD Phase 2)

## 1. 概述

這份文檔描述了貓咪乾物質計算器的第二階段功能開發計畫，包含首頁展示但尚未實作的進階功能。目標是將此應用從單純的營養計算工具擴展為全方位的貓咪生活管理平台。

## 2. 未實作功能清單

### 2.1 健康記錄管理

**優先級：P1（高）**

#### 功能描述
完整的醫療保健記錄系統，追蹤疫苗、體檢、用藥等重要健康資訊。

#### 詳細需求

**疫苗管理：**
- 疫苗種類記錄（三合一、五合一、狂犬病等）
- 接種日期與有效期限
- 下次接種提醒
- 疫苗證書照片上傳

**醫療記錄：**
- 診療日期與診所資訊
- 症狀描述與診斷結果
- 處方藥物與用法
- 醫療費用記錄
- 診療報告照片上傳

**健康指標追蹤：**
- 體重變化曲線
- 體溫記錄
- 食慾與活力評分
- 異常症狀記錄

### 2.2 飲食日記系統

**優先級：P1（高）**

詳細功能規劃請參考：[飲食日記系統 PRD](./2_飲食日記_PRD.md)

### 2.3 智能提醒系統

**優先級：P2（中）**

#### 功能描述
餵食、用藥、清潔、體檢等重要任務提醒，不再錯過照護時間。

#### 詳細需求

**餵食提醒：**
- 固定時間餵食提醒
- 特殊飲食時程提醒
- 斷食前提醒

**醫療提醒：**
- 用藥時間提醒
- 疫苗到期提醒
- 定期健檢提醒
- 驅蟲時程提醒

**日常照護提醒：**
- 貓砂清理提醒
- 梳毛時間提醒
- 指甲修剪提醒
- 洗澡時程提醒

### 2.4 支出管理系統

**優先級：P2（中）**

#### 功能描述
詳細記錄貓咪相關花費，包含飼料、醫療、用品等開支統計。

#### 詳細需求

**支出分類：**
- 飼料與零食費用
- 醫療與保健費用
- 日用品與玩具費用
- 美容與清潔費用
- 其他雜項費用

**統計分析：**
- 月度/年度支出統計
- 支出類別佔比分析
- 平均每日養育成本
- 預算設定與超支提醒

**發票管理：**
- 電子發票掃描與儲存
- 支出證明照片上傳
- 發票資訊自動解析

### 2.5 庫存追蹤系統

**優先級：P2（中）**

#### 功能描述
管理貓糧、貓砂、玩具等消耗品庫存，提前預警補貨時間。

#### 詳細需求

**庫存管理：**
- 物品分類管理（飼料、貓砂、藥品、玩具等）
- 庫存數量追蹤
- 保存期限管理
- 使用量記錄

**補貨提醒：**
- 低庫存警報
- 即期品提醒
- 自動補貨建議
- 採購清單生成

**消耗分析：**
- 使用量統計分析
- 消耗速度預測
- 成本效益分析

### 2.6 數據分析儀表板

**優先級：P3（低）**

#### 功能描述
每月自動生成詳細報表，分析健康趨勢、支出統計與照護品質。

#### 詳細需求

**健康分析：**
- 體重變化趨勢圖
- 營養攝取分析圖表
- 醫療頻次統計
- 健康評分計算

**行為分析：**
- 食慾變化趨勢
- 活動量分析
- 睡眠品質評估

**財務分析：**
- 養育成本趨勢
- 支出預測模型
- 成本優化建議

**照護品質評分：**
- 餵食規律性評分
- 醫療保健完整度
- 環境清潔維護度
- 綜合照護品質評估

## 3. 技術擴展需求

### 3.1 新增資料庫表結構

**健康記錄表 (HealthRecords)**
```sql
id: UUID (PK)
cat_id: UUID (FK)
record_type: String (vaccine/medical/checkup)
date: DateTime
title: String
description: Text
cost: Float
next_due_date: DateTime (可選)
attachments: JSON (照片URL陣列)
created_at: DateTime
```

**飲食記錄表 (FeedingRecords)**
*詳細結構請參考：[飲食日記系統 PRD](./2_飲食日記_PRD.md)*

**提醒設定表 (Reminders)**
```sql
id: UUID (PK)
user_id: UUID (FK)
cat_id: UUID (FK, 可選)
reminder_type: String
title: String
description: Text
schedule_time: Time
frequency: String (daily/weekly/monthly)
is_active: Boolean
created_at: DateTime
```

**支出記錄表 (Expenses)**
```sql
id: UUID (PK)
user_id: UUID (FK)
cat_id: UUID (FK, 可選)
category: String
amount: Float
description: Text
receipt_url: String (可選)
expense_date: Date
created_at: DateTime
```

**庫存管理表 (Inventory)**
```sql
id: UUID (PK)
user_id: UUID (FK)
item_name: String
category: String
current_stock: Float
unit: String
expiry_date: Date (可選)
minimum_threshold: Float
created_at: DateTime
updated_at: DateTime
```

### 3.2 新增功能模組

**通知系統：**
- 推播通知整合
- 郵件提醒系統
- 行事曆同步

**檔案上傳：**
- 照片上傳與壓縮
- OCR 文字識別（發票、處方箋）
- 雲端儲存管理

**數據分析：**
- 圖表視覺化庫
- 統計計算引擎
- 報表生成系統

## 4. 開發優先序規劃

### Phase 2A - 核心擴展功能 (3-4個月)
1. 健康記錄管理
2. 飲食日記系統（詳見專門 PRD）
3. 基礎提醒功能

### Phase 2B - 管理功能 (2-3個月)
1. 支出管理系統
2. 庫存追蹤系統
3. 進階提醒功能

### Phase 2C - 分析功能 (2個月)
1. 數據分析儀表板
2. 報表生成系統
3. AI 建議系統

- 每日活躍用戶數 (DAU)

- 健康記錄完整度

## 6. 風險評估

### 技術風險
- 數據量增長對性能影響
- 複雜查詢優化需求
- 檔案儲存成本控制

### 產品風險
- 功能複雜度增加
- 學習成本提升
- 用戶介面設計挑戰

### 營運風險
- 開發時程延長
- 維護成本增加
- 競爭對手快速追趕

此第二階段開發計畫將讓貓咪乾物質計算器從單一功能工具進化為全方位寵物照護平台，大幅提升用戶黏性與市場競爭力。
